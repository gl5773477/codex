# interview-assist 产物验证标准

> `--guard` 触发，验证 interview_report.md 质量。

## 验证目标

`job/{jd-title}/candidates/{name}/interview_report.md`

## 合格标准

### L0：结构完整性（硬性）
- [ ] 文件存在于正确路径
- [ ] 包含「基本信息」区块
- [ ] 包含「面试过程记录」区块（非空）
- [ ] 包含「综合评价」区块（含维度表 + Agent 建议）
- [ ] 包含「面试官最终决策」留白区块
- [ ] 顶部有版本标记（`版本：草稿`）

### L1：内容合法性（半硬性）
- [ ] 过程记录中每个话题有 🏷️ 考察点打标
- [ ] 综合评价维度表非空（≥ 1 行）
- [ ] 决策区无 Agent 填写内容（只有留白提示）
- [ ] 无模板占位符残留（`{xxx}` 未替换的变量）

### L2：质量标准（软性，Agent 判断）
- [ ] 过程记录忠实反映转录内容（无无中生有的内容）
- [ ] 打标维度与 JD 核心要求相关
- [ ] Agent 建议有具体依据，不是空泛描述
- [ ] 简历摘要 ≤ 3 行

## 不合格条件（任一即拒绝）
- 缺少「面试官最终决策」区块
- 决策区被 Agent 填写了内容
- 过程记录为空
- 综合评价无维度表

## 验证方式
手动：按上方清单逐项检查
`--guard` 时：Agent 读取报告文件，对照 L0-L2 逐项输出 ✅/❌
