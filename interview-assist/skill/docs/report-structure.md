# interview_report.md 产物结构规范

> 描述最终报告的结构标准。实际渲染以 `templates/report_template.md` 为准，本文件是结构契约。

## 文件路径

```
interview-assist-data/job/{jd-title}/candidates/{name}/interview_report.md
```

## 必须区块（顺序固定）

| 区块 | 来源 | 是否可为空 |
|------|------|---------|
| 基本信息 | 参数 + 留白 | ❌ |
| 简历摘要 | Phase 2，≤ 3 行 | ❌ |
| 面试过程记录 | Phase 3 | ❌ |
| 算法题评估 | Phase 3/4，可选 | ✅ 无代码题时省略 |
| 综合评价（维度表 + Agent 建议） | Phase 4 | ❌ |
| 面试官最终决策 | 留白 | ❌ 必须存在且为空 |

## 打标格式规范

过程记录中的打标：
```markdown
🏷️ **考察点**：[维度名称]（如：系统设计 / 代码能力 / 沟通表达）
📝 **面试官标注**：[原始关键词，不修改]
💬 **Agent 小结**：[1-2 句，基于回答内容的客观描述]
```

## 决策区格式（严格留白）

```markdown
---
## 面试官最终决策
> ⚠️ 以下内容请面试官填写，Agent 不填写任何判断。

**最终评价**：

**决定**：□ 通过　□ 拒绝　□ 待定

**备注**：
```

## 版本标记

报告顶部必须包含：
```
> 版本：草稿 | 生成时间：YYYY-MM-DD | 由 interview-assist 辅助生成
```
