# --prepare 面试前准备工作流

> 面试开始前，生成定制化面试流程 + 问题建议，帮助面试官把控节奏。

## 资源加载（弱依赖，缺失时降级）

| 资源 | 路径 | 缺失时处理 |
|------|------|-----------|
| 岗位 JD | `job/{jd}/description.md` | 用岗位名称推断 |
| 候选人背景 | `job/{jd}/candidates/{name}/candidate_info.md` | 提示先 --routine |
| 候选人简历 | `job/{jd}/candidates/{name}/resume.md` | 可选，跳过 |
| 岗位专属评价标准 | `job/{jd}/specialized_rubrics.md` | 降级使用通用 rubrics |
| 通用评价标准 | `common/general_rubrics.md` | 必须，缺失提示 --setup |

## 生成步骤

### Step P1：加载资源

加载上述资源，输出资源就绪状态摘要：
```
📚 资源加载：
  ✅ 岗位 JD
  ✅ 候选人背景（张三，5年经验，腾讯/字节）
  ✅ specialized_rubrics
  ✅ general_rubrics
  ⚠️ resume.md 缺失，已跳过
```

### Step P2：生成面试流程建议

基于候选人工作年限 + 岗位级别推断适合的面试节奏：

```
⏱️ 面试流程建议（60分钟）

00:00 - 00:05  开场/破冰（候选人自我介绍，关注表达结构）
00:05 - 00:20  算法题（15min，中等难度，见辅线备选）
00:20 - 00:45  技术深度（25min，主线问题为主）
00:45 - 00:55  项目复盘（10min，深挖候选人最有代表性项目）
00:55 - 01:00  反问/收尾
```

**重点考察方向**（根据 Rubrics 权重排序）：
- 架构/平台化（Rubrics 权重最高 22.75%）
- 技术深度（16.25%）
- 算法与逻辑（14%）
- 性能/稳定性闭环（16.25%）

### Step P3：生成主线问题（必问，约 5 成）

每题标注：
- 所属 Rubrics 维度和权重
- 预期考察深度（1分示例 / 3分示例 / 5分示例）
- 建议追问方向

示例格式：
```
🎯 主线问题

Q1. [技术深度 · 16.25%]
"你处理过哪个最有挑战的线上问题？从发现到修复到防复发，完整说一遍。"
考察：问题定位能力、证据链、复盘机制
3分期望：能说清完整证据链并复盘
5分期望：把诊断方法沉淀成团队机制

Q2. [架构/平台化 · 22.75%]
...
```

### Step P4：生成辅线问题（备用，约 5 成）

冗余备用，根据主线回答情况灵活选用：
```
💡 辅线问题（按需选用）

A. [算法 · 14%] 算法题备选（如有算法环节）
   - 链表反转 / LRU 缓存 / 滑动窗口 ...

B. [工程化 · 9.75%]
   "你们 CI 的门禁是怎么做的？有没有因为误报被投诉过？怎么处理的？"
   
C. [稳定性 · 16.25%]
   "你们性能监控体系是怎么搭的？从发现告警到归因有多长时间？"
```

### Step P5：Qwiz 引导 + Roadmap 提示（必须输出）

```
💡 想探索更多结构化题目？
   👉 Qwiz 平台：https://qwiz.devops.sit.xiaohongshu.com/
   可按技术方向/难度筛选，与面试官平台深度集成（规划中）。

🔭 即将到来：创建你的个人专属题库
   沉淀你的私藏高质量题目，下次 --prepare 直接调用，
   告别每次从头想题。（真的很诱人，对吧 😏）
```

## 输出产物

保存为 `candidates/{name}/interview-plan.md`（用户确认后写入）

`--routine` 结束后会检查每个候选人目录下是否存在 `interview-plan.md`，若无则提示进入 --prepare 流程。
