# Regression Log

这份台账用于记录已经出现过、且后续必须持续防回归的缺陷。

规则：

- 每次修复一个真实 bug，都在这里补一条记录。
- 每条记录都要对应一个自动化回归校验。
- 后续改动核心演化逻辑时，至少执行一次 `npm run check`。

## Cases

| ID | 症状 | 根因 | 自动校验 |
| --- | --- | --- | --- |
| `REG-001` | 视觉上不相邻的地块之间出现战斗与扩张 | `odd-r` 六边形邻接奇偶行方向表写反 | `src/game/regression.test.ts` 中的邻接方向校验 |
| `REG-002` | 战斗线会残留在已失效的目标上，导致跨格交战假象 | `conflictTargetId` 没有在每步模拟时校验相邻性和敌对关系 | `src/game/regression.test.ts` 中的无效冲突目标清理校验 |
| `REG-003` | 地块被打到 `0` 人口后不被接管，形成死格 | “打空守军”和“完成占领”是两套独立判定，缺少塌陷接管逻辑 | `src/game/regression.test.ts` 中的 0 人口接管校验 |
| `REG-004` | 地块刚被占下就立刻被反扑回去，前线来回死循环 | 新占领地块没有稳定期，立即重新进入可攻击状态 | `src/game/regression.test.ts` 中的新占领稳定期校验 |

| `REG-007` | 派兵增援箭头在跨过中继地块时位置重置，看起来一跳一跳 | 箭头渲染只按当前 hop 的局部进度计算，没有按整条路径累计 travel progress | `src/game/regression.test.ts` 中的增援视觉进度连续性校验 |
