# Territory Sandbox

一个以“上帝视角”为核心的大陆领地演化原型。

## 项目简介

`Territory Sandbox` 的目标不是立刻做成完整策略游戏，而是先验证一件事：

玩家站在世界之外，观察一块大陆上的多个势力自动扩张、争夺、吞并，这个过程本身是否足够有趣。

当前阶段聚焦“自动演化沙盘”这一最小闭环，优先保证：

- 世界会自己变化
- 势力会自己扩张
- 玩家能看懂局势变化
- 玩家可以做少量上帝式干预

## 当前玩法定位

玩家不扮演单个国家，而是作为整个世界的观察者与干预者存在。

当前版本只提供轻量交互：

- 暂停 / 继续演化
- 切换模拟速度（1x / 2x / 4x）
- 点击领地查看信息
- 对选中领地施加“神赐人口”

## 当前已实现内容

- 六边形网格地图
- 海洋 / 陆地基础生成
- 多势力初始分布
- 每块领地自动增长人口
- 人口达到阈值后自动攻击相邻敌方领地
- 战斗后占领并改色
- 势力排名面板
- 领地详情查看
- 少量白点作为人口活动的视觉表达

## 设计原则

- 优先做出“看着有意思”的自动演化，而不是先做复杂系统
- 默认沿用现有代码基线继续迭代，不另起一套
- 优先小步快跑，做最小可验证改动
- 所有后续增强都应尽量服务于三件事：
  - 更有观赏性
  - 更容易理解局势变化
  - 更方便做轻量干预

## 暂不优先实现

当前阶段默认不优先投入以下方向，除非明确提出：

- 复杂经济系统
- 建筑树 / 科技树
- 外交系统
- 英雄 / 角色养成
- 复杂单位寻路
- 细粒度实时战斗

## 本地运行

已验证可用环境：

- Node.js `v25.8.1`
- npm `11.11.0`

```bash
npm install
npm run dev
```

生产构建：

```bash
npm run build
```

说明：

- Vite 默认使用 `5173` 端口；如果端口已被占用，会自动切换到下一个可用端口。

## 技术栈

## 推荐的后续迭代方向

1. 让大陆形状与边界更自然
2. 增强 AI 扩张策略，让演化更有戏剧性
3. 增加更多上帝干预手段
4. 优化人口与战争的可视化表现
5. 逐步加入事件、特殊人物等轻量层次
